# 常见问题

汇总使用 MDOnline 过程中的常见问题及解决方案。

## 安装与启动

### Q: 启动后页面空白

**A:** 检查以下几点：

1. 确认访问地址正确（`http://localhost:8080`）
2. 确认在项目根目录启动了 HTTP 服务器
3. 直接用文件协议（`file://`）打开 `index.html` 可能无法正常工作，必须通过 HTTP 服务器访问

### Q: 端口被占用

**A:** 更换端口启动：

```bash
python -m http.server 3000
```

或在 Windows 上查看占用端口的进程：

```bash
netstat -ano | findstr :8080
```

### Q: `python` 命令不存在

**A:** 尝试使用 `python3` 或 `py`：

```bash
python3 -m http.server 8080
# 或
py -m http.server 8080
```

## 侧边栏问题

### Q: 侧边栏不显示新文档

**A:** 新增 `.md` 文件后需要重新生成侧边栏：

```bash
python gen_sidebar.py
```

然后刷新浏览器（`Ctrl+Shift+R` 强制刷新）。

### Q: 侧边栏链接文字显示为文件名

**A:** 文件缺少一级标题。确保 `.md` 文件的第一行是 `# 标题内容`，例如：

```markdown
# 我的文档标题

正文内容...
```

### Q: 侧边栏分组顺序不对

**A:** 修改 `gen_sidebar.py` 中的 `GROUP_ORDER` 配置：

```python
GROUP_ORDER = ['快速开始', '功能特性', '写作指南', '部署运维']
```

## 页面显示问题

### Q: 404 错误

**A:** 常见原因：

| 场景 | 原因 | 解决方案 |
|------|------|---------|
| 首页 404 | `homepage` 配置指向不存在的文件 | 修改 `index.html` 中的 `homepage` |
| 点击侧边栏 404 | 文件路径不匹配 | 检查 `_sidebar.md` 中的链接路径 |
| 部署后 404 | SPA 路由未回退 | 配置 Nginx/Vercel 的路由回退 |

### Q: 搜索功能不可用

**A:** 检查 `search.min.js` 文件是否存在且路径正确。在 `index.html` 中确认：

```html
<script src="./search.min.js"></script>
```

### Q: 主题切换按钮不显示

**A:** 检查 `style.css` 中是否包含主题切换按钮的样式定义。按钮由 `index.html` 中的插件动态创建。

## 部署问题

### Q: Nginx 部署后 `.md` 文件变成下载

**A:** Nginx 未配置 `.md` 的 MIME 类型。在 Nginx 配置中添加：

```nginx
types {
    text/plain  md;
}
```

### Q: 子路径部署后资源加载失败

**A:** 检查 Nginx 的 `alias` 指令末尾是否有 `/`：

```nginx
# 正确
location /apidocs/ {
    alias /var/www/mdonline/;
}

# 错误 — 缺少尾部斜杠
location /apidocs/ {
    alias /var/www/mdonline;
}
```

### Q: Vercel 部署后刷新页面 404

**A:** 在项目根目录创建 `vercel.json`：

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

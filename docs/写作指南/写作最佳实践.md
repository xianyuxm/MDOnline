# 写作最佳实践

本文提供在 MDOnline 中撰写文档的最佳实践和建议，帮助你写出清晰、一致、易维护的文档。

## 文件命名规范

| 规则 | 示例 | 说明 |
|------|------|------|
| 使用中文命名 | `入门指南.md` | 与侧边栏显示名一致 |
| 不使用空格 | ~~`入门 指南.md`~~ | 避免编码问题 |
| 不使用特殊字符 | ~~`入门#指南.md`~~ | 避免路由异常 |
| 文件名唯一 | — | 不同目录下也尽量不重名 |

## 标题层级规范

```markdown
# 文档标题          ← 每个 .md 文件必须有且仅有一个一级标题
## 章节            ← 文档的主要段落
### 小节            ← 章节内的细分内容
#### 细节           ← 极少使用，保持层级扁平
```

### 核心原则

1. **每个文件一个 `#` 标题**：`gen_sidebar.py` 读取一级标题作为侧边栏链接文字
2. **不要跳级**：`#` 之后直接跟 `##`，不要跳到 `###`
3. **层级不超过 4 级**：如果内容太深，考虑拆分为多个文件
4. **标题简洁有力**：用名词短语而非完整句子

## 内容组织

### 单文件原则

一个 `.md` 文件应聚焦一个主题。判断拆分时机：

- 文件超过 300 行 → 考虑拆分
- 出现多个并列的二级标题 → 可拆为子文档
- 不同角色关注不同部分 → 按角色拆分

### 文档模板

推荐每个文档遵循以下结构：

```markdown
# 文档标题

一句话概述本文档的内容。

## 概述（可选）

背景信息和上下文。

## 正文

### 步骤/要点一

具体内容...

### 步骤/要点二

具体内容...

## 常见问题（可选）

## 相关文档

- [相关文档A](/docs/xxx/xxx)
- [相关文档B](/docs/xxx/xxx)
```

## 代码示例规范

### 标注语言类型

始终在代码块中标注语言，以启用语法高亮：

```markdown
<!-- 推荐 -->
```python
def hello():
    print("Hello")
```

<!-- 不推荐 -->
```
def hello():
    print("Hello")
```
```

### 代码应可运行

提供的代码示例应当是完整的、可运行的，而不是片段：

```markdown
<!-- 推荐：完整可运行 -->
```python
import requests

response = requests.get("https://api.example.com/status")
print(response.status_code)
```
```

## 链接规范

### 使用相对路径

```markdown
<!-- 推荐 -->
[入门指南](/docs/快速开始/入门指南)

<!-- 不推荐：硬编码域名 -->
[入门指南](https://example.com/docs/快速开始/入门指南)
```

### 链接文字要有意义

```markdown
<!-- 推荐 -->
阅读[安装与部署指南](/docs/快速开始/安装与部署)了解详情

<!-- 不推荐 -->
点击[这里](/docs/快速开始/安装与部署)了解详情
```

## 图片使用建议

- 图片存放在 `docs/` 下的 `images/` 或 `assets/` 子目录
- 使用相对路径引用：`![截图](./images/screenshot.png)`
- 控制图片宽度：大图影响加载速度和阅读体验
- 为图片提供替代文字：便于无障碍访问

## 版本标注

当文档内容与特定版本相关时，明确标注版本号：

```markdown
> 本文档适用于 MDOnline v2.0 及以上版本
```

## 更新维护

- 新增文档后运行 `python gen_sidebar.py` 更新侧边栏
- 删除文档后同样需要重新生成侧边栏
- 修改文档一级标题后重新生成侧边栏以更新链接文字

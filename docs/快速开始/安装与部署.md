# 安装与部署

MDOnline 是纯静态站点，可以部署到任何支持静态文件托管的平台。以下是几种常见的部署方式。

## 本地开发

### 启动预览服务器

**方式一：双击启动（推荐）**

双击 `start-server.bat`，脚本会自动生成侧边栏并启动服务器。

> 如果双击运行失败，请使用方式二。

**方式二：手动启动**

在项目根目录打开命令行，手动执行：

```bash
py gen_sidebar.py          # 生成侧边栏（可选，_sidebar.md 已存在时可跳过）
py -m http.server 8080     # 启动服务器
```

> 如果 `py` 命令不可用，请尝试 `python` 或 `python3`。

### 修改端口

默认端口为 `8080`，如需修改：

```bash
python -m http.server 3000
```

## Nginx 部署

### 基础配置

```nginx
server {
    listen 80;
    server_name docs.example.com;
    root /var/www/mdonline;

    location / {
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

### 子路径部署

如果需要部署在 `/apidocs/` 子路径下：

```nginx
server {
    listen 80;
    server_name example.com;

    location /apidocs/ {
        alias /var/www/mdonline/;
        index index.html;
        try_files $uri $uri/ /apidocs/index.html;
    }
}
```

> MDOnline 的 `basePath` 会自动检测子路径，无需手动修改配置。

### MIME 类型配置

Nginx 默认可能不会为 `.md` 文件设置正确的 MIME 类型，需添加：

```nginx
types {
    text/plain  md;
}
```

## GitHub Pages 部署

1. 将项目推送到 GitHub 仓库
2. 进入仓库 **Settings → Pages**
3. Source 选择 `main` 分支，目录选 `/ (root)`
4. 保存后等待部署完成

## Vercel 部署

1. 在 Vercel 控制台导入 Git 仓库
2. Framework Preset 选择 **Other**
3. Output Directory 保持默认
4. 点击 Deploy

> Vercel 默认会为 `.md` 文件设置正确的 MIME 类型，无需额外配置。

## 部署检查清单

- [ ] 侧边栏是否正确生成（运行 `python gen_sidebar.py`）
- [ ] `.md` 文件 MIME 类型是否为 `text/plain`
- [ ] SPA 路由是否回退到 `index.html`
- [ ] 站点图标 `favicon.svg` 是否正常加载
- [ ] 全文搜索是否可正常使用
